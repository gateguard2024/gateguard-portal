# Intern Day 1 Plan - Nexus Beta Test

## Goal

The intern is not being tested. Nexus is being tested.

The intern's job is to use Nexus like a brand-new user and report what is easy, confusing, broken, missing, or slow.

## Rule

If the intern has to think too hard, that is a product failure we need to capture.

Use this format for every issue:

```txt
I was trying to:
I clicked:
I expected:
What happened:
Was it confusing, broken, missing, or slow?
```

## 4-5 Hour Schedule

### 0:00-0:20 - Welcome and Setup

1. Log into beta.
2. Open Nexus.
3. Confirm access to:
   - My Day
   - Sales
   - Jobs
   - Customers/Sites
   - Money/Docs
   - Internal
4. Write first impression:
   - What do you think this software does?
   - Which button would you click first?
   - Which words are unclear?

### 0:20-1:00 - First Use Test

No coaching.

Tasks:

1. Open My Day and answer: What should I handle today?
2. Open Sales and answer: Where would I add a lead?
3. Open Jobs and answer: Where are jobs needing attention?
4. Open Customers/Sites and answer: Where do I find a customer/property?
5. Open Money/Docs and answer: Where would I find paperwork or invoices?
6. Open Internal and answer: Where would I report a bug?

Score each section:

```txt
Easy / Confusing / Broken / Missing
```

### 1:00-1:45 - Customers/Sites Test

Test:

- Find Customer
- Find Property
- Properties Needing Attention
- Property Systems

Tasks:

1. Search for a customer if test data exists.
2. Search for a property if test data exists.
3. Click a result.
4. Open the overview.
5. Report whether the overview makes sense.

If no data exists, write:

```txt
No data found. Empty state was clear / confusing.
```

### 1:45-2:15 - Break and Notes Cleanup

Clean up notes using:

```txt
Issue:
Where:
What I expected:
What happened:
How serious:
```

Severity:

- High = blocks me
- Medium = confusing but I can continue
- Low = looks wrong or wording issue

### 2:15-3:00 - Money/Docs Test

Test:

- Invoices
- Renewals
- Documents to Sign
- Compliance

For each board, answer:

1. Can I tell what this board is for?
2. Can I tell what needs attention?
3. Can I click an item and understand it?

### 3:00-3:45 - Internal Tracker / PWA Test

Open Internal -> Tracker.

Check buckets:

- Today
- Blocked
- Bugs
- Next Up

PWA/mobile test:

1. Open Nexus in PWA/mobile.
2. Switch tabs.
3. Open a glass board.
4. Close the board.
5. Refresh/reopen app.
6. Report any navigation, layout, reload, or modal issues.

### 3:45-4:30 - Lead Loading Practice

Go to Sales.

Try to add or understand how to add a lead.

A lead should include:

- Company/property name
- Contact name
- Email
- Phone
- Address if known
- Notes
- Next step

Answer:

1. Could I tell where to add a lead?
2. Could I tell what information was required?
3. Could I find the lead again?
4. Could I tell whether it needed a quote?

### 4:30-5:00 - End of Day Summary

Use this format:

```txt
Top 3 things that were easy:
1.
2.
3.

Top 3 things that were confusing:
1.
2.
3.

Top 3 things that felt broken:
1.
2.
3.

Words I did not understand:
1.
2.
3.

My recommendation:
The first thing I would fix is:
```

## Success Criteria

By the end of Day 1, we should know:

- What is easy
- What is confusing
- What is broken
- What data needs to be loaded first
- What must be fixed before deeper beta testing

# Intern Setup Checklist

Use this before the intern starts beta testing.

## 1. User Invite

Yes, the intern should be invited as a real beta user.

Minimum setup:

- Name
- Email
- Role: beta tester / internal tester / limited admin as appropriate
- Organization: Gate Guard internal org
- Access: Nexus, Sales, Jobs, Customers/Sites, Money/Docs, Internal Tracker

Do not give production admin access on Day 1 unless required.

## 2. My Day Tasks

Yes, create Day 1 tasks for him after the user exists.

Suggested My Day tasks:

1. Open Nexus and write first impression
2. Test My Day, Sales, Jobs navigation
3. Test Customers/Sites search and overview
4. Test Money/Docs boards
5. Test Internal Tracker
6. Test PWA/mobile
7. Practice adding one lead
8. Complete end-of-day summary

## 3. Internal Tracker Items

Create tracker items for anything found during testing.

Buckets:

- Today
- Blocked
- Bugs
- Next Up

Issue types to use in title/body:

- PWA bug
- Confusing UX
- Missing data
- Workflow gap
- Looks wrong
- Broken action

Example title:

```txt
PWA bug - bottom nav hard to use after opening glass board
```

## 4. Data Needed for Testing

Load at least one sample of each if possible:

- Lead
- Contact
- Company/customer
- Property/site
- Quote/estimate
- Job/work order
- Invoice
- Renewal/contract dates
- Document signature record

If the database is empty, empty-state testing is still useful.

## 5. Printed Materials

Print:

- Intern Day 1 Plan
- First Impression Sheet
- Section Scorecard
- Issue Report Forms
- Lead Loading Sheet
- PWA/Mobile Test Sheet
- End of Day Summary

## 6. Pass/Fail Rule

If the intern needs coaching to find the right area, the workflow is not simple enough.

Capture the confusion. Do not explain it away.
